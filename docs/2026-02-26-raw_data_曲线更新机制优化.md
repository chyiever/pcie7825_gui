## 2. 需求概述

### 2.1 问题背景
- 当前Raw数据模式下GUI卡顿严重，特别是点击STOP后容易死机
- 数据传输量过大（40-80MB/次），导致内存和CPU资源紧张
- FFT计算频繁，阻塞主线程
- 显示逻辑复杂，影响用户体验

### 2.2 优化目标
- 减少数据传输量99%以上（从40MB降至320KB）
- 降低GUI更新频率，提高稳定性
- 实现多线程FFT计算，避免主线程阻塞
- 简化显示逻辑，区分单/双通道模式

### 3.2 显示逻辑优化

#### 单通道模式
- **时域图**：第一个子图显示4帧平均后的单条曲线
- **频域图**：第二个子图显示FFT结果（可选）
- **更新机制**：时域1秒，FFT3秒

#### 双通道模式
- **时域图**：第一、二个子图分别显示各通道的4帧平均曲线
- **频域图**：禁用FFT功能
- **更新机制**：仅时域1秒更新

def _read_raw_data(self):
    # 读取完整数据
    points_per_ch = self._total_point_num * self._frame_num
    data, _ = self.api.read_data(points_per_ch, self._channel_num)

    # 仅传输前4帧给GUI（关键优化点）

### 4.4 GUI显示逻辑重构

**文件**: `src/main_window.py`

#### 核心显示方法
```python
def _update_raw_display(self, data: np.ndarray, channel_num: int):
    """重构的Raw数据显示方法"""

    # 1秒间隔控制
    current_time = time.time()
    if (current_time - self._last_time_domain_update) < 1.0:
        return

    if channel_num == 1:
        # 单通道：4帧平均显示，可选FFT
        averaged_frame = self._compute_averaged_frame(data, point_num)
        self.plot_curve_1[0].setData(averaged_frame)  # 显示完整分辨率

        # FFT处理（3秒间隔）
        if (spectrum_enabled and fft_interval_met):
            self.fft_worker.calculate_fft(data[:point_num])

    elif channel_num == 2:
        # 双通道：分别显示，禁用FFT
        for ch in range(2):
            averaged_frame = self._compute_averaged_frame(data[:, ch])
            self.plot_curve_1[ch].setData(averaged_frame)  # 显示完整分辨率
```

#### 通道控制逻辑
```python
def _on_channel_changed(self, index: int):
    """通道切换时的控制逻辑"""
    channel_num = self.channel_combo.currentData() or 1

    if data_source in [DataSource.RAW, DataSource.AMPLITUDE]:
        is_single_channel = (channel_num == 1)

        # 双通道时禁用FFT
        self.spectrum_enable_check.setEnabled(is_single_channel)
        self.psd_check.setEnabled(is_single_channel)

        if not is_single_channel:
            self.spectrum_enable_check.setChecked(False)
            self.plot_widget_2.setVisible(False)
```
